# Protocol

Six operations define how a conformant implementation interacts with `theta.toml`. The [manifest](../manifest/index.md) defines the format; the protocol defines the operations. For normative requirements, see the [protocol spec](../spec/protocol.md).

## Operations

| Operation | Input | Output | Network | Description |
|---|---|---|---|---|
| [**Validate**](conformance.md#validation) | `theta.toml` | Diagnostics | No | Structural and semantic validation |
| [**Lock**](sources.md#lockfile) | `theta.toml` | `theta.lock` | **MAY** (git fetch) | Resolve all dependencies, pin versions |
| [**Materialize**](materialization.md) | `theta.lock` | `.theta/` | No | Populate materialization directory from locked sources |
| [**Cast to**](casting.md) | `theta.toml` + `.theta/` | Harness config files | No | Convert manifest to harness-native format |
| [**Cast from**](import.md) | Harness config files | `theta.toml` + source files | No | Import harness config into a manifest |
| [**Check**](conformance.md#check) | `theta.toml` + `theta.lock` + `.theta/` | Diagnostics | No | Full pipeline validation |

## Protocol artifacts

| Artifact | Lifecycle | Description |
|---|---|---|
| `theta.toml` | User-authored | The manifest — **MUST** be committed to version control |
| `theta.lock` | Generated by lock | Pinned dependency graph — **MUST** be committed to version control |
| `.theta/` | Generated by materialize | Materialization directory — **SHOULD** be gitignored |
